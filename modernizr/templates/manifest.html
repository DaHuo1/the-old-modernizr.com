Modernizr

The Modernizr Manifest

Modernizr is a JavaScript library that uses feature detection to identify native support in the current User Agent (a.k.a. web browser), then adds classes to the document's <body> element and also creates a matching JavaScript object that reports what features are natively available for designing the page with.

Modernizr was made with three distinct goals in mind:

  • To discourage and diminish the practice of sniffing User Agent strings so prevalent among web developers today;
  • To give web designers and developers the opportunity to reliably use new CSS 3 and HTML 5 features without the fear of losing control in browsers that don't support these features yet;
  • To encourage broad adoption of these new features which, in turn, gives browser vendors more feedback to improve their support and establish these upcoming Web Standards.

Through more widespread adoption of these cutting-edge features from CSS 3 and HTML 5, the community will discover many more bugs and performance issues than the browser vendors on their own would, which means a win-win situation: browsers become better and faster, and users get better and better experiences the more we designers & developers start using these technologies.

Web Standards

The best way to push the Web forward as a platform is to adopt and encourage web standards-based development practices. Innovation on the Web is not stifled by these standards, in fact, great innovation can be made alongside web standards just fine. But the present situation on the Web is that many websites are designed for the lowest common denominator—either IE6 or IE7 for most companies—and as a result, progress happens only slowly, with only few websites taking advantage of new features available in some browsers today.

Creating workarounds for some browsers can be cumbersome and supporting multiple versions of a site is often undesirable; with Modernizr, it is no longer cumbersome to create a workaround (if desired), and maintaining multiple "looks" is almost a non-issue entirely. Modernizr set out to make it much easier than ever before to really use new features and still have full control over your website in browsers that don't yet support them.

Web Standards can only be considered successful if they are adhered to. As such, Modernizr explicitly limits its feature-detection tests to methods that are standards compliant. If a standards-compliant test produces a false positive in any of the main browsers, the test is scrapped until a different—but still standards-compliant—method is found. For example, text-shadow is not part of Modernizr's current release because all standards-compliant tests produce false positives in Firefox 2 and 3.0.

Performance

Another major requirement was performance. Modernizr is designed to run in under 20ms on the page, depending on the speed of the host system, and a firm requirement was that only one HTTP request would be issued (HTTP requests have a significant impact on page performance).

While it is typically preferred that JavaScript includes are added to the end of a page so as to prevent the rendering from being held up by the loading of the script file, Modernizr is expressly included right after the opening tag of the body element. By its very nature, Modernizr is very likely to influence what style rules end up being applied to the page, which means that it should be executed and completed before the browser starts rendering anything. This causes a small performance impact but ends up safeguarding the user experience from unnecessary flickering in the page.
